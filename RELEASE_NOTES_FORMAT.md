# Release Notes Format Guide

Write release notes on each GitHub release using this format so they render
correctly inside the DATO Toolkit update dialog.

## Template

```
## What's New

- Description of new feature or fix
- Another change

## Bug Fixes

- Fixed issue where X caused Y
- Corrected Z behavior

## Notes

Any important notes about this release (breaking changes, migration steps,
known issues).
```

## Tips

- Use `##` for section headings (renders as bold headers in the dialog)
- Use `- ` for bullet points
- Use `**text**` for bold emphasis
- Keep each bullet to one line if possible
- Always include a "What's New" section
